## กติกาหลักจาก QM-QMR-001-1 ที่ระบบบังคับใช้

- **ระดับเอกสาร 4 ระดับ**: QM (คู่มือคุณภาพ) → SOP (แนวทางปฏิบัติ) →
  WI (วิธีปฏิบัติงาน) → FM (แบบฟอร์ม) + เอกสารภายนอก (EXT)
- **ระบบรหัส `AAA-BBB-XXX-YY`** — ระดับ-หน่วยงาน-ลำดับที่-ครั้งที่แก้ไข
  ระบบออกรหัสให้อัตโนมัติเมื่ออนุมัติ (เช่น `SOP-PTC-001-1`)
- **รหัสหน่วยงานจริง 29 หน่วย** — QMR, HRD, RMC, ICC, PCT, PTC, ENV, IM,
  CHC, DEN, PHA, LAB, RAD, REH, NUT, HIS, ADM, PHC, TCM, PSY, MSO, NSO,
  OPD, IPD, ER, OR, IC, LR, HD
- **ตารางผู้มีอำนาจ** จัดทำ/ตรวจสอบ/อนุมัติ แตกต่างตามระดับเอกสาร
  (ผู้แจกจ่าย/เรียกคืน/ทำลาย = QMR ทุกระดับ)
- **การยกเลิก**: ประทับตรา "ยกเลิก" เก็บต้นฉบับ 1 ปีนับจากวันยกเลิก
  แล้วจึงทำลาย — ระบบคำนวณวันครบกำหนดให้

## ระบบล็อกอินและสิทธิ์ผู้ใช้ (RBAC)

ล็อกอินด้วย **อีเมล + รหัส OTP 6 หลัก** (โหมดสาธิตโชว์รหัสบนจอ พร้อมสลับไปใช้
Firebase Auth email link/OTP ได้) จำ session ไว้ใน `localStorage` แบ่งผู้ใช้ 4 ระดับ
พร้อมบังคับสิทธิ์ตามตารางผู้รับผิดชอบใน QM-QMR-001-1:

| ระดับผู้ใช้ | ตรวจสอบ | อนุมัติ | แจกจ่าย | เมนูที่เห็น |
|---|---|---|---|---|
| **ผู้อำนวยการโรงพยาบาล** | — | QM/SOP/WI/FM (ทุกระดับ) | ✓ | ครบทุกหน้า |
| **ประธานกก./หัวหน้ากลุ่มงาน** | — | WI/FM | — | ทุกหน้ายกเว้นการแจกจ่าย |
| **คณะกรรมการศูนย์คุณภาพ** | ✓ | — | ✓ | ครบทุกหน้า |
| **ผู้ใช้ทั่วไป** | — | — | — | แดชบอร์ด/บัญชี/ยื่นคำขอ/กล่องรับ/คู่มือ |

ปุ่มและเมนูที่ไม่มีสิทธิ์จะถูกซ่อนหรือปิด และมีการกันเส้นทาง (route guard)
บัญชีสาธิต: `director@` · `qmr@` · `chair@` · `staff@paihospital.go.th`

## ผู้ช่วยข้อมูลยา (AI) — เว็บแยก `/drugai/`

เว็บสแตนด์อโลนสำหรับบุคลากร (แยกจากระบบเอกสาร QMR ไม่ต้องเข้าสู่ระบบ)
สร้างด้วย Generative AI เลือกผู้ให้บริการได้เหมือนสตูดิโอ มี 4 โหมด:

- **ข้อมูลยา** — monograph ย่อ: ข้อบ่งใช้ ขนาดยา ข้อห้าม ผลข้างเคียง
  ข้อควรระวัง ตั้งครรภ์/ให้นมบุตร พร้อมป้ายยาความเสี่ยงสูง (High-Alert)
- **ตรวจยาตีกัน** — วิเคราะห์ drug interaction รายคู่ พร้อมระดับความรุนแรง
  และคำแนะนำการจัดการ (ชื่อที่ AI ไม่รู้จักจะถูกเตือนแยกไว้ ไม่นับเป็นผ่าน)
- **คำแนะนำผู้ป่วย** — ร่างฉลากคำแนะนำการใช้ยาภาษาง่าย พร้อมคัดลอกไปใช้
- **ถาม-ตอบเรื่องยา** — คำถามอิสระ เช่น ความคงตัว การปรับขนาดในผู้ป่วยไต

> ข้อมูลจาก AI เป็นข้อมูลอ้างอิงเบื้องต้น ต้องผ่านการตรวจทานโดยเภสัชกร
> ก่อนนำไปใช้กับผู้ป่วยจริงทุกครั้ง

ซอร์สอยู่ที่ `drugai/index.html` + `src/drugai/` (Vite multi-page ใน repo เดียวกัน
build/deploy พร้อมกันอัตโนมัติ) และใช้ไลบรารี AI ร่วมกับสตูดิโอ (`src/lib/ai.ts`,
`src/lib/drugAi.ts`)

## จัดตารางเวร & ค่าตอบแทนนอกเวลาราชการ — เว็บแยก `/schedule/`

เว็บสแตนด์อโลนสำหรับจัดเวรบุคลากรและออก **ใบเบิกค่าตอบแทนการปฏิบัติงานนอกเวลาราชการ**
ของกลุ่มงาน (ออกแบบตามงานเภสัชกรรม รพ.ปาย แต่รองรับหลายหน่วยงาน/หลายวิชาชีพ) แยกจากระบบ QMR
ไม่ต้องเข้าสู่ระบบ บันทึกข้อมูลไว้ในเบราว์เซอร์ (`localStorage`):

- **ปฏิทินจัดเวรรายเดือน** — คลิกแต่ละวันเพื่อจัดสรรบุคลากรลงกะ (เช้า/บ่าย/Oncall/พิเศษ)
  ไฮไลต์เสาร์–อาทิตย์และวันหยุดนักขัตฤกษ์ กรองตามหน่วยงาน และเลื่อนเดือนก่อน–ถัดไป
- **จัดตารางอัตโนมัติ** — สร้างเวรจาก “วันเวรประจำสัปดาห์” ของแต่ละคน โดยกะ Oncall ให้
  ผู้ที่ตั้งเป็น Oncall Only, วันหยุด→เวรเช้า, วันราชการ→เวรบ่าย และไม่สร้างซ้ำของเดิม
- **สรุปค่าตอบแทน** — ตารางแจกแจงจำนวนกะ/ชั่วโมงและยอดเงินรายคน (อัตราต่อชั่วโมงแยกตาม
  วิชาชีพ, Oncall คิดข้ามเที่ยงคืน) พร้อม **ส่งออก Excel 3 ชีต** (สรุปเวร / ค่าตอบแทน /
  บัญชีลงเวลา) ด้วย `exceljs` — จัดหน้า A4 ฟอนต์ TH Sarabun New จำนวนเงินเป็นตัวอักษรไทย
- **ตั้งค่าระบบ** — ข้อมูลโรงพยาบาล, ทะเบียนบุคลากร (คำนำหน้า/ชื่อเล่น/สี/วิชาชีพ/สังกัด/
  วันเวรประจำสัปดาห์), วันหยุด (ตัวคูณ 1/1.5/2), หน่วยงานย่อย, อัตราเบิกจ่าย 10 วิชาชีพ,
  ช่วงเวลากะ, ธีม/ขนาดฟอนต์ และ **ลายมือชื่อ** (วาดบน canvas หรืออัปโหลด) ของผู้จัดทำ/ผู้อนุมัติ
- **พิมพ์ / บันทึก PDF** — ปฏิทินและรายงานมีเลย์เอาต์เฉพาะสำหรับพิมพ์ (`print-only`/`no-print`)

ซอร์สอยู่ที่ `schedule/index.html` + `src/schedule/` (Vite multi-page — build/deploy
พร้อมกันอัตโนมัติ) มีสโตร์/สถานะของตัวเองแยกจากระบบ QMR หน้าสรุป (ExcelJS) โหลดแบบ lazy
เฉพาะเมื่อเปิดใช้

## เทคโนโลยี

- **Frontend**: React 19 + TypeScript + Vite 6 + Tailwind CSS v4 + Motion + Lucide
- **State**: in-memory store ผ่าน `useSyncExternalStore` (โหมดสาธิต — รีเฟรชแล้วรีเซ็ต)
- **Backend (option)**: Express + Nodemailer (`/api/notify`) สำหรับแจ้งเตือนอีเมล
- **AI (เลือกผู้ให้บริการได้)**: Claude (`@anthropic-ai/sdk`), OpenAI, หรือ Google Gemini
  — เลือกเจ้าและใส่ API key ของเจ้านั้นในหน้าสตูดิโอหรือหน้าข้อมูลยา (เก็บแยกกันใน localStorage) โหมดสาธิต
  เรียกจากเบราว์เซอร์ด้วย key ของผู้ใช้ (ควรย้ายไปฝั่งเซิร์ฟเวอร์เมื่อใช้จริง)
- **แอปจัดตารางเวร**: `exceljs` (ออกใบเบิกค่าตอบแทน Excel 3 ชีต) + `motion` + `lucide`
  — โหลดหน้าสรุป/ExcelJS แบบ lazy เฉพาะเมื่อใช้งาน, เก็บสถานะทั้งหมดใน `localStorage`
- **พร้อมต่อยอด**: Firebase (Firestore/Storage) เปิดใช้เมื่อเซ็ต `VITE_FIREBASE_*`
